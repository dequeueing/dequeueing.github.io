## GPU 硬件基础 {#gpu-hardware}

GPU的主要构成：Streaming Multiprocessor。一块 GPU 包括多个 SM。

![GPU architecture](/images/blog/cuda1/gpu-architecture.png)

### Streaming Multiprocessor

一个 SM 有四个划分（partition），每个划分有 32 个核心（core），因此，一个 SM 包含了 128 个核心。

每个partition 有独立的寄存器（register）和 cache。

![SM architecture](/images/blog/cuda1/sm-arch.png)

## GPU Programming Model {#gpu-programming-model}

### Concepts
首先需要理解几个概念：

- Host：CPU，运行常规程序（C 代码、Python 代码）
- Device： GPU，运行cuda 内核代码。
- CUDA：GPU 编程驱动，一部分运行在 host 上，一部分运行在 device 上。

数据需要在 host 和 device 之间进行传输。

![Data movement](/images/blog/cuda1/data-movement.png)

### SIMT

GPU 运行的关键在于**并行**。术语为 SIMT (single instruction, multiple threads)，即多个线程同时执行一条指令。

GPU 的编程抽象（即程序员应该如何看 GPU）：

- *Thread Block*: 多个线程属于一个 thread block
- *Grid*：多个 block 属于一个 grid

**CUDA kernel is exeucted as grid of blocks of threads.**

### Programming Model 与 硬件的 mapping 

这个subsection主要解释 programming model 和 GPU 硬件之间的对应关系，以及每个线程（thread）具体是如何在 GPU 上运行起来的。

#### Warp

Warp 是GPU 中线程的最小执行单元。Streaming Multiprocessor partitions a thread block into *warps*, which is the unit of GPU creating, managing, scheduling and executing. 


> 如何理解最小执行单元？
同一个 warp 当中的线程，共享PC 和其他寄存器，在同一个时钟周期中执行相同的 instruction。但可以独立 branch。

每个 warp 包括 32 个线程（因此可理解为每个 warp 就是在 SM 的一个 partition上执行，见下图）

![Warp](/images/blog/cuda1/warp.png)


> 可以将 warp 类比做操作系统的进程，是资源调度和管理的单位。


## CUDA Kernel {#cuda-kernel}

每个 kernel 都是一个在 GPU 上运行的程序。程序的编码本身是线性的（serial），但是执行是并行的，由多个线程同时执行。因此，kernel 正确编写的关键在于，**每个 thread 负责自己相对应的数据。**

### Compilation

nvcc是 CUDA 的编译器（类似g++至于 c++）。一个 CUDA 程序中包括host code和device code，分别进行编译。

![CUDA compilation](/images/blog/cuda1/cuda-compile.png)

> nvcc -o output.so --shared src.cu -Xcompiler -fPIC

注意这个指令，它的意思是编译的产物是一个 shared library，因此它无法被直接当做一个完整的二进制文件运行，而是需要被其他程序（C 或者 Python 程序）加载并运行。这部分后续会有更详细的说明。

### Key Words


| Key Words | Call on | Execute on |
|----------|----------|----------|
| \_\_global\_\_  | host   | device   |
| \_\_device\_\_  |  device | device   | host  |
|  \_\_host\_\_  | host  | host   | host   |


### Build-in Variable 

因为CUDA编程是由 grid of block of thread 来构成的，因此需要能够精准定位到每一个线程。由此有三个内置的变量：

- gridDim
- blockDim
- blockIdx
- threadIdx

这些内置变量需要在 kernel 启动时进行配置。kernel 以如下方式启动：

```
kernelFuncName<<<Dg, Db>>>(args)
```

其中 Dg 和Db 分别是 grid 和 block 的维度。**Dg and Db are either dim3 or int。**

----
一个 CUDA kernel 的例子：

```
__global__ void VecAddKernel(int* A, int* B, int* C, int n) {
  int i = blockDim.x * blockIdx.x + threadIdx.x;
  if (i < n) {
    C[i] = A[i] + B[i];
  }
}

```


启动一个CUDA kernel：


```
// n: the size of the vector
int n = 1024;
int threads_per_block = 256;
int num_blocks = (n + threads_per_block - 1) / threads_per_block;
VecAddKernel<<<num_blocks, threads_per_block>>>(dA, dB, dC, n);
```

在这个例子当中，grid 和 block 的类型都是 int，因此grid 和 block 的维度都是 1。


## CUDA Examples {#cuda-examples}

### Vector Addition

```c
__global__ void VecAddKernel(int* A, int* B, int* C, int n) {
  // blockDim is size of block along x-axis
  // blockIdx is the index of the current thread's block
  // threadIdx is the index of the current thread within the block
  // Compute the global thread ID
  int i = blockDim.x * blockIdx.x + threadIdx.x;
  if (i < n) {
    // Calculate the addition of the ith element of A and B
    C[i] = A[i] + B[i];
  }
}
```

> int i = blockDim.x * blockIdx.x + threadIdx.x;

回到那个关键的问题：每个 thread 应该负责哪一部分元素的处理。在 vector addition 这个例子当中，每一个 thread负责一个 element 即可。前提是 thread 的数量应大于向量的长度。


运行该 kernel：

```c
void VecAddCUDA2(int* Acpu, int* Bcpu, int* Ccpu, int n) {
  // In this example, we load the data into the GPU by C++ codes.
  int *dA, *dB, *dC;
  // Allocate device memory
  cudaMalloc(&dA, n * sizeof(int));
  cudaMalloc(&dB, n * sizeof(int));
  cudaMalloc(&dC, n * sizeof(int));

  // Copy data from host memory to device memory
  cudaMemcpy(dA, Acpu, n * sizeof(int), cudaMemcpyHostToDevice);
  cudaMemcpy(dB, Bcpu, n * sizeof(int), cudaMemcpyHostToDevice);

  // Launch the CUDA kernel
  int threads_per_block = 256;
  int num_blocks = (n + threads_per_block - 1) / threads_per_block;
  VecAddKernel<<<num_blocks, threads_per_block>>>(dA, dB, dC, n);

  // Copy the result from device memory to host memory
  cudaMemcpy(Ccpu, dC, n * sizeof(int), cudaMemcpyDeviceToHost);


  // Free device memory
  cudaFree(dA); 
  cudaFree(dB); 
  cudaFree(dC);
}

```

>    `int num_blocks = (n + threads_per_block - 1) / threads_per_block;`

注意这里我们确保了线程的数量是足够的，足以处理完向量中的所有元素。


### Matrix Multiplication

这里我们呈现两种 kernel 的方法，关键区别在于dimension 的维度。

首先看 dimension 为 1 的情况：

```c
__global__ void MatmulKernel(const float* a, const float* b, float* out, 
                             int M, int N, int P) {
  // Calculate the global thread index and the row and column it corresponds to
  // Every thread will compute one element of the output matrix
  int idx = threadIdx.x + blockIdx.x * blockDim.x;
  int row = idx / P;
  int col = idx % P;
  // Compute the summation of the dot product of the row of a and the column of b
  if (row < M && col < P) {
    float sum = 0.0;
    for (int i = 0; i < N; i++) {
      sum += a[row * N + i] * b[i * P + col];
    }
    out[row * P + col] = sum;
  }
}

```

> int idx = threadIdx.x + blockIdx.x * blockDim.x;

这里thread block 和 grid 都是一维的，因此我们需要额外计算出该 element 需要计算的 row 和 column。

接下来是 dimension 为 2 的情况：

```c
__global__ void matrixMul(const int *a, const int *b, int *c, int M, int N, int P) {
  // Compute each thread's global row and column index
  int row = blockIdx.x * blockDim.x + threadIdx.x;
  int col = blockIdx.y * blockDim.y + threadIdx.y;
  if (row >= M || col >= P) return;
  // Iterate over row, and down column
  c[row * P + col] = 0;
  for (int k = 0; k < N; k++) {
    // Accumulate results for a single element
    c[row * P + col] += a[row * N + k] * b[k * P + col];
  }
}
```

因为这里 block 是二维的，因此可以通过内置变量直接得到 row 和 column 的值。

尽管 dimension 划分不同，相同的地方在于，每个线程都需要负责一个output element 的计算，因此每次计算都需要同时遍历矩阵 A 的一行以及矩阵 B 的一列。



## Run CUDA code in Python {#run-cuda-code-in-python}

为了能够在 Python 中调用 cuda 程序，首先需要将CUDA 程序编译为 shared library：

```
nvcc -o output.so --shared src.cu -Xcompiler -fPIC
```

其中 `--shared` flag 就代表编译产物是共享 binary，可以被其他程序加载。

在 Python code 中，需要使用`pycuda`库。

demo 如下：

```python
import ctypes
import pycuda.gpuarray as gpuarray
import pycuda.driver as cuda
from pycuda.compiler import SourceModule
import pycuda.autoinit

# Load the shared library
cur_path = os.getcwd()
lib = ctypes.CDLL(os.path.join(cur_path, "matmul.so"))

# Define argtypes and returntypes of the C function
lib.Matmul.argtypes = [
    ctypes.POINTER(ctypes.c_float),
    ctypes.POINTER(ctypes.c_float),
    ctypes.POINTER(ctypes.c_float),
    ctypes.c_int,
]

# Call the C wrapper function with CUDA kernel
lib.Matmul(
    ctypes.cast(a_gpu.ptr, ctypes.POINTER(ctypes.c_float)),
    ctypes.cast(b_gpu.ptr, ctypes.POINTER(ctypes.c_float)),
    ctypes.cast(c_gpu.ptr, ctypes.POINTER(ctypes.c_float)),
    ctypes.c_int(m),
    ctypes.c_int(n),
    ctypes.c_int(p)
)
```

注意：上述代码忽略了将创建和转移数据的相关代码，只保留了加载共享库和运行 cuda kernel 的代码。

## Practice 1: Map Operation CUDA Kernel {#practice-1-map-operation-cuda-kernel}

Implement the CUDA kernel for element-wise map operations and integrate it with the MiniTorch framework. The map operation applies a unary function to every element of an input tensor, producing a new tensor with the same shape. For example, applying f(x) = x² to tensor [1, 2, 3] yields [1, 4, 9].

思路：每个 thread 负责一个 element。

### Helper Functions

张量（Tensor）是深度学习中常见的数据类型，为了完成这些 practice，系统提供了一系列的 helper functions，这对我们理解 tensor 很重要。

#### index_to_position

```c
__device__ int index_to_position(const int *index, const int *strides, int num_dims)
{
  /**
   * Converts a multidimensional tensor index into a single-dimensional position in storage
   * based on strides.
   * Args:
   *    index: index tuple of ints
   *    strides: tensor strides
   *    num_dims: number of dimensions in the tensor, e.g. shape/strides of [2, 3, 4] has 3 dimensions
   *
   * Returns:
   *    int - position in storage
   */
  int position = 0;
  for (int i = 0; i < num_dims; ++i)
  {
    position += index[i] * strides[i];
  }
  return position;
}
```

这个 helper function 的作用是，给定一个 tensor 中的 index，以及该 tensor 的步长（strides），确定它在 memory storage 当中的 position。

> 首先，什么是 stride？

定义是：当前维度每增加 1，会跳过多少个元素。比如，对于一个 **compact** 的tensor 来说，若它的 shape 是[2,3]，那么它的 strides 就是[3，1]，因为在第一个维度每次加一就会跳过 3 个元素，而在第二个维度每次加 1 会跳过一个。

> 为什么会出现 stride 和 shape 不匹配的情况？

这是由于 tensor 在逻辑上和在内存存储上会出现不一致的情况，比如 view() 只取前三列，但是view()得到的新 tensor 并不会分配新的地址空间，因此 stride 与 shape 不匹配。

图解index_to_position这个 helper function：

![Index to position](/images/blog/cuda1/index-to-position.png)

#### to_index

```c
__device__ void to_index(int ordinal, const int *shape, int *out_index, int num_dims)
{
  /**
   * Convert an ordinal to an index in the shape. Should ensure that enumerating position 0 ... size of
   * a tensor produces every index exactly once. It may not be the inverse of index_to_position.
   * Args:
   *    ordinal: ordinal position to convert
   *    shape: tensor shape
   *    out_index: return index corresponding to position
   *    num_dims: number of dimensions in the tensor
   *
   * Returns:
   *    None (Fills in out_index)
   */
  int cur_ord = ordinal;
  for (int i = num_dims - 1; i >= 0; --i)
  {
    int sh = shape[i];
    out_index[i] = cur_ord % sh;
    cur_ord /= sh;
  }
}
```


整体上是跟`index_to_position`刚好相反。


![Position to index](/images/blog/cuda1/to_index.png)

这里需要理解的是这一行代码：

```
out_index[i] = cur_ord % sh;
```

需注意到cur_ord一直在被整除，所以cur_ord代表的是去除掉已经考虑过的 dimension，还可以被分配到当前 dimension 的 element 数量。

#### broadcast_index

```c

__device__ void broadcast_index(const int *big_index, const int *big_shape, const int *shape, int *out_index, int num_dims_big, int num_dims)
{
  /**
   * Convert a big_index into big_shape to a smaller out_index into shape following broadcasting rules.
   * In this case it may be larger or with more dimensions than the shape given.
   * Additional dimensions may need to be mapped to 0 or removed.
   *
   * Args:
   *    big_index: multidimensional index of bigger tensor
   *    big_shape: tensor shape of bigger tensor
   *    shape: tensor shape of smaller tensor
   *    out_index: multidimensional index of smaller tensor
   *    num_dims_big: number of dimensions in bigger tensor
   *    num_dims: number of dimensions in smaller tensor
   *
   * Returns:
   *    None (Fills in out_index)
   */
  for (int i = 0; i < num_dims; ++i)
  {
    if (shape[i] > 1)
    {
      out_index[i] = big_index[i + (num_dims_big - num_dims)];
    }
    else
    {
      out_index[i] = 0;
    }
  }
}
```

> 什么是 broadcast？

broadcast是指，当两个 dimension 不同的 tensor 要发生相互作用时（比如相加，或者一个 tensor 是另一个 tensor 的函数结果），shape 相对更小的 tensor 会将自身的元素广播到自身没有的那个维度上。

![Broadcast](/images/blog/cuda1/broadcast.png)


num_dims_big - num_dims就是两个 tensor 相差的维度。

### Practice 1 Solution

```c

__global__ void mapKernel(
    float *out,
    int *out_shape,
    int *out_strides,
    int out_size,
    float *in_storage,
    int *in_shape,
    int *in_strides,
    int shape_size,
    int fn_id)
{
  /**
   * Map function. Apply a unary function to each element of the input array and store the result in the output array.
   * Optimization: Parallelize over the elements of the output array.
   *
   * You may find the following functions useful:
   * - index_to_position: converts an index to a position in a compact array
   * - to_index: converts a position to an index in a multidimensional array
   * - broadcast_index: converts an index in a smaller array to an index in a larger array
   *
   * Args:
   *  out: compact 1D array of size out_size to write the output to
   *  out_shape: shape of the output array
   *  out_strides: strides of the output array
   *  out_size: size of the output array
   *  in_storage: compact 1D array of size in_size
   *  in_shape: shape of the input array
   *  in_strides: strides of the input array
   *  shape_size: number of dimensions in the input and output arrays, assume dimensions are the same
   *  fn_id: id of the function to apply to each element of the input array
   *
   * Returns:
   *  None (Fills in out array)
   */

  int out_index[MAX_DIMS];
  int in_index[MAX_DIMS];

  /// BEGIN HW1_1
  /// TODO
  // Hints:
  // 1. Compute the position in the output array that this thread will write to
  int i = blockDim.x * blockIdx.x + threadIdx.x;
  if (i < out_size) {
    // 2. Convert the position to the out_index according to out_shape
    to_index(i, out_shape, out_index, shape_size);

    // 3. Broadcast the out_index to the in_index according to in_shape (optional in some cases)
    broadcast_index(out_index, out_shape, in_shape, in_index, shape_size, shape_size);


    // 4. Calculate the position of element in in_array according to in_index and in_strides
    int in_pos = index_to_position(in_index, in_strides, shape_size);

    // 5. Calculate the position of element in out_array according to out_index and out_strides
    int out_pos = index_to_position(out_index, out_strides, shape_size);

    // 6. Apply the unary function to the input element and write the output to the out memory
    out[out_pos] = fn(fn_id, in_storage[in_pos]);

    // assert(false && "Not Implemented");
    /// END HW1_1
  } 
}
```

这里比较 tricky 的设置在于，考虑到了小 tensor 广播到大 tensor 的情况，并且 input 和 output 的 shape 与 strides 均可能不对应，因此需要利用 helper function 进行处理。


## Practice 2: Zip Operation CUDA Kernel {#practice-2-zip-operation-cuda-kernel}

Implement the CUDA kernel for element-wise zip operations and integrate it with the framework. This operation applies a binary function to corresponding elements from two input tensors, producing a new tensor with the same shape. For example, applying addition f(x,y) = x + y to tensors [1, 2, 3] and [4, 5, 6] yields [5, 7, 9].

这道 practice 与 practice 1 非常类似，输入从一个 tensor 变成两个 tensor，因此 CUDA kernel 并无太大的差别。

Practice 2 还需要将 CUDA kernel integrate 到 Python 代码当中：

```c
extern "C"
{
  void tensorZip(
      float *out,
      int *out_shape,
      int *out_strides,
      int out_size,
      int out_shape_size,
      float *a_storage,
      int *a_shape,
      int *a_strides,
      int a_size,
      int a_shape_size,
      float *b_storage,
      int *b_shape,
      int *b_strides,
      int b_size,
      int b_shape_size,
      int fn_id)
  {
    // Allocate device memory
    float *d_out, *d_a, *d_b;
    cudaMalloc(&d_a, a_size * sizeof(float));
    cudaMalloc(&d_b, b_size * sizeof(float));
    cudaMalloc(&d_out, out_size * sizeof(float));

    int *d_out_shape, *d_out_strides, *d_a_shape, *d_a_strides, *d_b_shape, *d_b_strides;
    cudaMalloc(&d_out_shape, out_shape_size * sizeof(int));
    cudaMalloc(&d_out_strides, out_shape_size * sizeof(int));
    cudaMalloc(&d_a_shape, a_shape_size * sizeof(int));
    cudaMalloc(&d_a_strides, a_shape_size * sizeof(int));
    cudaMalloc(&d_b_shape, b_shape_size * sizeof(int));
    cudaMalloc(&d_b_strides, b_shape_size * sizeof(int));

    // Copy data to the device
    cudaMemcpy(d_a, a_storage, a_size * sizeof(float), cudaMemcpyHostToDevice);
    cudaMemcpy(d_b, b_storage, b_size * sizeof(float), cudaMemcpyHostToDevice);
    cudaMemcpy(d_out_shape, out_shape, out_shape_size * sizeof(int), cudaMemcpyHostToDevice);
    cudaMemcpy(d_out_strides, out_strides, out_shape_size * sizeof(int), cudaMemcpyHostToDevice);
    cudaMemcpy(d_a_shape, a_shape, a_shape_size * sizeof(int), cudaMemcpyHostToDevice);
    cudaMemcpy(d_a_strides, a_strides, a_shape_size * sizeof(int), cudaMemcpyHostToDevice);
    cudaMemcpy(d_b_shape, b_shape, b_shape_size * sizeof(int), cudaMemcpyHostToDevice);
    cudaMemcpy(d_b_strides, b_strides, b_shape_size * sizeof(int), cudaMemcpyHostToDevice);

    // Launch kernel
    int threadsPerBlock = 32;
    int blocksPerGrid = (out_size + threadsPerBlock - 1) / threadsPerBlock;
    zipKernel<<<blocksPerGrid, threadsPerBlock>>>(
        d_out, d_out_shape, d_out_strides, out_size, out_shape_size,
        d_a, d_a_shape, d_a_strides, a_shape_size,
        d_b, d_b_shape, d_b_strides, b_shape_size,
        fn_id);

    // Copy back to the host
    cudaMemcpy(out, d_out, out_size * sizeof(float), cudaMemcpyDeviceToHost);
    cudaDeviceSynchronize();

    // Check CUDA execution
    cudaError_t err = cudaGetLastError();
    if (err != cudaSuccess)
    {
      fprintf(stderr, "Zip Error: %s\n", cudaGetErrorString(err));
      exit(EXIT_FAILURE);
    }

    // Free memory on device
    cudaFree(d_a);
    cudaFree(d_b);
    cudaFree(d_out);
    cudaFree(d_out_shape);
    cudaFree(d_out_strides);
    cudaFree(d_a_shape);
    cudaFree(d_a_strides);
    cudaFree(d_b_shape);
    cudaFree(d_b_strides);
  }
}

```

注意，上面这段代码依旧是CUDA 代码的一部分。其中`extern "C"`是在告诉编译器，作用域中的函数是给其他语言的代码调用的。

### Basic

这里我们补充跟 Cuda shared memory 相关的基础知识。

#### CUDA Shared Memory 

两个基础原语：
- `__shared__` 用来声明一个 block -内所有线程共享的高速内存；
- `__syncthreads()` 用来让同一个 block 内的线程在某个位置同步。

shared memory 只对当前 block 当中的线程可见，一个 block 当中的所有线程都可以对这一块贡献内存进行读写。由于不同线程的执行进度可能不同，因此需要一个 barrier 在合适指令处对所有线程进行同步，以确保shared memory 对所有线程是一致的。

一个例子：

```c
__global__ void reduce0(int *g_idata, int *g_odata) {
    __shared__ int sdata[];
    int pos = threadIdx.x;
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    sdata[tid] = g_idata[i];
    __syncthreads();
    for(int s = 1; s < blockDim.x; s *= 2) {
        if (tid % (2 * s) == 0) {
            sdata[tid] += sdata[tid+s];
        }
        __syncthreads();
    }
    if (tid == 0) {
        g_odata[blockIdx.x] = sdata[0];
    }
}
```

#### Solution

综合考虑，这道题目需要使用 shared memory 用于暂存输入张量中某个维度的部分和，然后通过 parallel reduction 的方式，得到最终reduction 的结果，完整代码如下：

```c

    float *out,
    int *out_shape,
    int *out_strides,

    int *a_shape,
    int *a_strides,
    int reduce_dim,
    float reduce_value,
    int shape_size,
    int fn_id)
{
  /**
   * Reduce function. Apply a reduce function to elements of the input array a and store the result in the output array.
   * Optimization:
   * Parallelize over the reduction operation. Each kernel performs one reduction.
   * e.g. a = [[1, 2, 3], [4, 5, 6]], kernel0 computes reduce([1, 2, 3]), kernel1 computes reduce([4, 5, 6]).
   *
   * You may find the following functions useful:
   * - index_to_position: converts an index to a position in a compact array
   * - to_index: converts a position to an index in a multidimensional array
   *
   * Args:
   *  out: compact 1D array of size out_size to write the output to
   *  out_shape: shape of the output array
   *  out_strides: strides of the output array
   *  out_size: size of the output array
   *  a_storage: compact 1D array of size in_size
   *  a_shape: shape of the input array
   *  a_strides: strides of the input array
   *  reduce_dim: dimension to reduce on
   *  reduce_value: initial value for the reduction
   *  shape_size: number of dimensions in the input & output array, assert dimensions are the same
   *  fn_id: id of the reduce function, currently only support add, multiply, and max
   *
   *
   * Returns:
   *  None (Fills in out array)
   */

  // __shared__ double cache[BLOCK_DIM]; // Uncomment this line if you want to use shared memory to store partial results
  int out_index[MAX_DIMS];
  int in_index[MAX_DIMS];

  int i = blockIdx.x;
  int tid = threadIdx.x;
  int thread_num = 32;

  // Debug print
  // if (i == 0) {
  //   printf("Printing a starage value: \n");
  //   for (int j = 0; j < out_size * a_shape[reduce_dim]; j += 1) {
  //     printf("%f ", a_storage[j]);
  //   }
  //   printf("\nNumber of elements in a: %d\n", out_size * a_shape[reduce_dim]);
  // }

  if (i < out_size) {
    // 2. Convert the position to the out_index according to out_shape
    to_index(i, out_shape, out_index, shape_size);

    // 3. Initialize the reduce_value to the output element
    // Note: Since out is compact, so out_pos has the same value as i
    // int out_pos = index_to_position(out_index, out_strides, shape_size);  
    double inter = reduce_value;

    // 4. Iterate over the reduce_dim dimension of the input array to compute the reduced value
    // Idea: copy the output_index, and then iteratively change the reduce_dim value

    // 4.1 Init in_index first 
    for (int j = 0; j < shape_size; j+= 1) {
      in_index[j] = out_index[j];
    }

    // Declare shared memory
    __shared__ float sdata[32];
    sdata[tid] = reduce_value;
    for (int j = tid; j < a_shape[reduce_dim]; j += thread_num) {
      in_index[reduce_dim] = j;
      int in_pos = index_to_position(in_index, a_strides, shape_size);
      sdata[tid] = fn(fn_id, sdata[tid], a_storage[in_pos]);
    }
    __syncthreads();

    // Parallel add the elements in shared memory 
    for(int s = 1; s < thread_num; s *= 2) {
        if (tid % (2 * s) == 0) {
            // sdata[tid] += sdata[tid+s];
            sdata[tid] = fn(fn_id, sdata[tid], sdata[tid + s]);
        }
        __syncthreads();
    }


    // 5. Write the reduced value to out memory
    if (tid == 0) { 
      out[i] = sdata[0];
    }
    
  } 
}
```

## Practice 4: Matrix Multiplication {#practice-4-matrix-multiplication}

Implement the CUDA kernel for matrix multiplication and integrate it with the framework. This is one of the most important operations in deep learning and offers significant opportunities for optimization.

矩阵乘法是 cuda 计算中相当重要的一个操作，是众多深度学习算子的不可缺少的一部分。矩阵乘法最简单的做法，就是每一个元素负责 output 中的一个 element。这里我们要介绍的是tiling 这种优化技巧。图示如下：

![Tiling](/images/blog/cuda1/tiling.png)

tiling 的关键点在于减少对 global memory 的访问，而把计算一片区域所需的数据先存放在 shared memory，由于 shared memory 距离 processor 更近，因此有更快的带宽。

> 来自Programming Massively Parallel Processors的分析：
With 16 * 16 tiles, one can reduce the global memory accesses by a factor of 16. This increases the compute to global memory access ratio from 0.25 OP/B to 4 OP/B. This improvement allows the memory bandwidth of a CUDA device to support a higher computation rate. For example, in the A100 GPU which has a global memory bandwidth of 1555 GB/second, this improvement allows the device to achieve (1555 GB/second) (4 OP/B)=6220 GFLOPS, which is substantially higher than the 389 GFLOPS achieved by the kernel that did not use tiling.

分块方式：每一个 block 负责一个 tile，每一个 thread 负责计算 tile 当中的一个元素。不断滚动 tile 并且累加便得到最终的结果。

关键代码如下：

加载数据到 shared memory 当中：

```c
for (int tile_index = 0; tile_index < (n + TILE - 1) / TILE; tile_index += 1) {
// index of A and B to read from 
int col_a = tile_index * TILE + tx, row_a = row;
int row_b = tile_index * TILE + ty, col_b = col;

// Convert to position
int pos_a =
    batch * a_batch_stride +
    row_a * a_strides[1] +
    col_a * a_strides[2];

int pos_b =
    batch * b_batch_stride +
    row_b * b_strides[1] +
    col_b * b_strides[2];

// Load data into shared memory
if (row_a < m && col_a < n) {
  a_shared[ty][tx] = a_storage[pos_a];
} else {
  a_shared[ty][tx]  = 0;
}
if (row_b < n && col_b < p) {
  b_shared[ty][tx]  = b_storage[pos_b];
} else {
  b_shared[ty][tx]  = 0;
}

__syncthreads();  

```

累加进行求和：
```c
for (int k = 0; k < TILE; k++) {
  inter += a_shared[ty][k] * b_shared[k][tx];
}
__syncthreads();  
```

这里有两个易错点：
1. 边界检查。一些 thread 的 index 可能会超出矩阵的范围，此时需要进行一次边界判断。
2. index 计算。需要明确tx, ty的含义。比如：

```c
dim3 gridDims(
  (p + threadsPerBlock - 1) / threadsPerBlock,
  (m + threadsPerBlock - 1) / threadsPerBlock,
  batch
);
```

这里，一个 grid 的 x 是p（矩阵的行数），y 是m（矩阵的列数）。那么在确定 tiling 的时候就需要小心：

```c
int bx = blockIdx.x, by = blockIdx.y;
int row = TILE * by + ty;
int col = TILE * bx + tx;
```

tx 和 ty 也是同理：需要明确它们的含义，从而计算出正确的坐标。
