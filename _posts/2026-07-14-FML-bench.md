## Introduction {#introduction}

一些常见的 agent reserach strategy：
1. greedy-hill climbing (autoresarch from andrej)
2. parallel linear idea chains
3. through tree research 
4. evolutionary optimization

关键问题：strategy 的什么关键因素会决定reserach 的结果？

需要的 benchmark，应该区分：
1. agent strategy (how agent generate ideas, learn from previous experiemnts...)
2. execution infra (code editor, exepriemtn runner, val/test sepration)

另外，需要提供一系列的process-level metrics，从而能够根据内部情况进行成果的归因。

![FML-bench overview](/images/blog/fml-bench/fml-bench-overview.png)

得到的三个 finding：
1. complexity 跟performance 是无关的。比如autoresearch versus TAS, 尽管两个 strategy 的 complexity 不同，但是最终 performance 类似。
2. greedy search 在 improvement opportunity 更密集的任务上更有效。所谓improvement opportunity，是指能否通过少量修改，获得不错的提升。
3. performance 的决定因素：early convergence and direcionally focused exploration. 其他的一些因素则关联性不大。

感觉：findings 不够 systematic。缺少更加 systematic 的 reserach question。但也可以理解，回归到这个 paper 想要研究的：区别不同的 strategy的好坏。但是似乎还是只能从 benchmark 上面得到一些信息。

## FML-bench {#fml-bench}

Setup:
A single validation run completes within 40 mins. So a full 100-step agent run finishes within a few days.
**很明显，这里忽略了一个 strategy 的不同run 是有 variance 的。**


Evaluation pipeline: 


![Evaluation pipeline](/images/blog/fml-bench/pipeline.png)

1. task specification
2. Loop:
a. propose idea
b. edit code
c. run experiment 

3. step budget 耗尽之后，choose the best-validated codebase and run a single held-out evaluation, recording the best outcome and intermediate metrices.**(这个地方是很值得学习的)**

### Final performance metrices

1. normalized test improvement. 
2. pairwise win-rate. 

### (Important) Process-level metrices

1. exploration: 搜索的广度。用 embedding 去量化。

![Exploration metric](/images/blog/fml-bench/exploration.png)


2. generalization：validation 和 test 之间的gap。
3. reliability: 有多少个 step 是产生了有效的结果的。
4. efficiency：gain 随 step 的变化。
5. cost：number of tokens.

### Agent Implementation


Base class: 


```python
class BaseAgent(ABC):
    """Abstract base class for all AI research agents.

    Provides shared infrastructure for step counting, best-code tracking,
    code snapshot/restore, and token usage aggregation. Subclasses implement
    ``initialize()`` and ``run()`` with their own search/decision strategy.
    """

    def __init__(self, config: AgentConfig):
      pass

```


一个继承 BaseAgent 的类只需要实现`initialize()`和`run()`，这个类包括了大部分 code infra 的代码，比如：

```python
# ------------------------------------------------------------------
# Shared code-editing helper
# ------------------------------------------------------------------

def _edit_code(self, instruction: str, error_context: Optional[str] = None) -> bool:
    """Shared code editing via CodeEditor. Returns True if edit succeeded.

    Raises:
        RuntimeError: If ``self.editor`` has not been set (i.e., ``initialize()``
            was not called or did not assign an editor).
    """
    if self.editor is None:
        raise RuntimeError(
            "self.editor is None. Call initialize() before _edit_code()."
        )
    self._last_edit_instruction = instruction  # Full instruction for per-step logging
    result = self.editor.edit(instruction, error_context)
    self._last_edit_result = result
    if result.token_usage:
        self.token_usage_log.append(result.token_usage)
    return result.success

```

> 看起来 code editing 也是通过 LLM 的 api call 来实现的。


下面来看具体的一个 agent class:

```python
class AutoresearchAgent(BaseAgent):
    """
    Autoresearch: greedy hill-climbing ML research agent.

    Search policy (from program.md):
      1. Propose experimental idea (LLM call)
      2. Edit code (CodeEditor)
      3. Run experiment (_execute_val)
      4. If improved -> keep. If not -> discard. If crash -> debug or skip.
      5. Repeat.
    """

```

3. 进入main loop。
4. 在main loop中运行测试，并且构建最终结果。

```python
while self.budget_remaining():
```

取决于当前的状态，debug或者产生新idea：
```python
if self._pending_debug:
    # Debug retry: restore snapshot, edit with error context
    self._pending_debug = False
    self._restore_snapshot(self.current_snapshot)
    instruction = self._build_debug_instruction(
        self._last_error, self._last_idea
    )
    edit_ok = self._edit_code(instruction, error_context=self._last_error)
    idea = f"[debug retry] {self._last_idea}"
    action = "debug"
else:
    # Generate new idea via LLM
    idea = self._generate_idea()
    self._last_idea = idea
    # Restore to current best before editing
    self._restore_snapshot(self.current_snapshot)
    # Edit code via CodeEditor
    edit_ok = self._edit_code(idea)
    action = "draft" if self.step_count == 0 else "improve"
```

### Benchmark Runner

```python
        runner = BenchmarkRunner("cider", agent)

    """

```
