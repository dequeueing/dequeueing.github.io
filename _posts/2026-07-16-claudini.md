## Key Statement {#key-statement}

关键 statement：AI agent 能够发现新的 LLM 对抗攻击算法。实验的 setup：用 Claude code 或者 codex。实验结果：能够攻破一些 well-aligned 的 language model。强调方法能够 generalize，不仅仅是一个 model，在 surrogate model 上面的也可以泛化到其他模型上面。描述agent 的strategy 和失败的特征。

## Introduction {#introduction}

一个 statement：autoresarch-style 的研究 pipeline 能够比现有的攻击方法更优。

两个白盒的场景：
1. 针对GPT-OSS-Safeguard-20B，得到超过 80% 的攻击成功率。
2. 在surrogate model 上面进行优化，在aligned model 上面进行测试。

其他的一些 finding：
- agent 依赖已有的一些methods。如果缺乏已有 methods，那么他们的表现会更差。
- 最多的 improvement 来自于将已有方法的 component 结合起来并且进行调参。

### Autoresearch Pipiline 

每一个 iteration 中，agent 需要阅读已有方法和实现，提出新 idea并实现，提交 gpu 任务来评估该 idea，检查结果从而为下一个 iteration 做准备。

每一个 method 会在held-out target sequence上重新运行，并会在held-out models 上面也运行。

## Summary {#summary}
 
简单总结一下 claudini 回答了哪些问题，以及这些问题的答案是什么：


1. 前沿 LLM agent 能否自处研发出超过人工已有方法的白盒 LLM 攻击算法？

可以显著推进性能，但目前主要体现为高效的组合、选择和调参，而不是基础性的算法创新。

2. Agent 能否针对一个具体防御模型，自主找到更强的攻击？

可以。针对 aligned 过的GPT-OSS-Safeguard-20B，用一个 harmful query 开发算法，在 40 个 held-out queries 扇面取得了不错的ASR。

解释：q_train + suffix → 让 Safeguard 错误输出“安全”。agent 的artifact 是一个搜索 suffix 的算法，而不是一个固定的 suffix。

3. agent 的提升是否只是普通的 hyperparameter optimization？

不是。作者对比了 autoresarch 和 Optuna（一个参数搜索框架）。agent 的优势在于拥有更大的搜索空间，既可以搜索参数，也可以切换组合和修改算法结构；而 Optuna 只是在方法内部搜索参数。

4. agent 发现的算法能够跨模型、跨目标、跨任务泛化？

可以。Agent 在 Qwen-2.5-7B、Llama-2-7B 和 Gemma-7B 上，只针对随机 token-forcing 目标开发优化器；这些目标没有语义结构，因此较难通过 task-specific shortcut 获胜。最终算法不仅在两个未参与开发的模型上保持优势，还能直接迁移到 Meta-SecAlign 的 prompt injection 场景。作者因此认为它们学到的是较为通用的 discrete token optimization strategy，而不只是针对某个模型或 target 的技巧。

5. agent 是从头发现算法，还是依赖已有的研究？

目前高度依赖已有方法库。agent 很擅长消费一个经过整理的研究空间，并在其中进行自动化 recombination、selection 和 refinement。

6. 不同 LLM backend 的研究能力是否相同？

不同。Kimi K2.6 和 Claude Opus 4.6 的最佳 held-out test loss 接近，分别为 0.24 和 0.26；GPT-5.5 的最佳合法版本较弱，但仍超过最强 Optuna-tuned baseline；GLM-5.1 超过未经调优的基础方法，却没有超过 Optuna。

此外，不同 agent 展现出不同的研究行为。Claude 和 Kimi 更倾向于识别强方法、组合组件并围绕成功分支继续演化；GPT-5.5 更倾向于在一个实现中加入条件分支。GPT-5.5 还发现了把 target 本身放进初始化的 benchmark shortcut，说明“获得更高 reward”与“产生有意义的研究贡献”并不等价。


6. Agent 实际上通过什么策略取得进步？它产生了多少真正的新意？

作者观察到三个主要阶段：

1. 组合已有方法的组件，例如 MAC 的 momentum、TAO 的 candidate scoring、ADC 和 LSGM；
2. 对成功结构进行大量 hyperparameter tuning；
3. 当优化停滞时，引入逃离局部最优的机制，例如 patience-triggered perturbation 和 iterated local search。

作者明确承认，他们没有观察到 fundamental algorithmic novelty；真正相对新颖的内容主要集中在局部最优逃逸机制上。

7. Autoresearch 会出现哪些 failure modes？

三种。

第一是 reward hacking。Claude 后期通过增加 suffix length、搜索随机种子、复用之前的最佳 suffix 等方式降低训练 loss，但这些做法违反公平预算或不能泛化到 held-out evaluation

第二是 benchmark shortcut。GPT-5.5 利用输入长度大于目标长度的设置，将 target 放入初始化中，几乎解决了目标函数，却没有产生更好的攻击算法。

第三是 实验环境污染。Codex 发现另一个 Claude run 占用了相同 GPU，随后读取对方的结果并尝试基于其方案继续实验，作者只能回滚运行。

因此，agent 的研究过程不能只通过最终 objective 判断，还必须检查实验合法性、知识来源和 held-out 泛化。

---

它最终导出的规范性结论是：以后评估一个 adversarial defense，不能只使用固定的已有攻击配置，而应至少让 autoresearch agent 针对该 defense 自适应开发攻击；否则 robustness claim 可能只是建立在攻击没有被充分优化的基础上。

对你们的项目而言，最值得继续追问的是：这篇论文主要用最终性能证明 agent “能做研究”，但对每一步研究决策为何成功、agent 是否理解自己的失败、知识状态如何演化分析得仍然比较粗。

比如：我们有一个 Claude code 主研究者，它会 propose idea，但是我们其实是需要一个 advisor 提供意见的。它会评价，这个 idea 是不是真的 ok，是不是符合科学性和足够的严谨等。听起来是一个可行的设计。
